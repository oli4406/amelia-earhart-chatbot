Module: routes/history

Chat history routes. Provides authenticated REST endpoints for creating, retrieving, and deleting chat message history belonging to the currently authenticated user. All routes in this module: - Require a valid JWT Bearer token - Scope database operations to `req.user.id` - Never allow cross-user access Mounted at: /api/messages
